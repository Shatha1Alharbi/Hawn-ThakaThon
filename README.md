#  منصة هَـون (Hawn) 

### **وصف المشروع**
**هَـون** هي منصة رقمية مبتكرة قائمة على تقنيات الذكاء الاصطناعي، تهدف إلى ردم الفجوة المعرفية بين الطبيب والمريض. تقوم المنصة بتحليل نصوص الحوارات والزيارات الطبية المعقدة واستخلاص المعلومات الجوهرية منها، ثم إعادة صياغتها في ملخص مهيكل ومنظم بصرياً يسهل استيعابه.

يساعد التطبيق المرضى (خاصة ذوي الأمراض المزمنة وكبار السن) على استيعاب خططهم العلاجية بدقة، مما يساهم في تحسين الالتزام بالأدوية وتقليل الأخطاء الناتجة عن نسيان تعليمات الطبيب أو خلط المهام العلاجية.

---

### **المميزات التقنية**
* **(LLM):** استخدام نموذج **Noha 2.0** من شركة **عِلم** لتحليل السياق الطبي باللغة العربية بذكاء عالٍ.
* **التصنيف الآلي للمعلومات:** تمييز دقيق بين التشخيصات، الأدوية، التعليمات، ومواعيد المتابعة وتوزيعها في واجهة بطاقية (Card-based UI).
* **تجربة مستخدم متميزة (UI/UX):** تصميم عصري يعتمد على مبدأ "Glassmorphism" لضمان راحة المستخدم البصرية وسهولة الوصول للمعلومة.
* **أدوات التصدير والمشاركة:** دعم كامل لنسخ الملخصات المنظمة أو طباعتها وتصديرها كملفات **PDF**.

---

### **المعمارية التقنية (Architecture)**
يعتمد المشروع على معمارية **Client-side API Integration** لضمان السرعة والكفاءة:
* **Frontend:** HTML5, CSS3, Vanilla JavaScript.
* **AI Engine:** ELM Noha 2.0 (LLM).
* **Deployment:** Vercel (Static Hosting).

---

### **طريقة التشغيل**

يمكنك استعراض المشروع وتشغيله بإحدى الطرق التالية:

#### **1. التشغيل السحابي (Live Demo)**
يمكنك الوصول للمنصة مباشرة عبر الرابط الرسمي على Vercel:
[https://hawnthakathon.vercel.app](https://hawnthakathon.vercel.app)

#### **2. التشغيل المحلي (Local Setup)**
في حال رغبتك في تشغيل المشروع على جهازك الشخصي، اتبع إحدى الطريقتين:

* **الطريقة السريعة:**
    1. قم بتحميل المجلد وفك الضغط عنه.
    2. افتح ملف `index.html` مباشرة باستخدام أي متصفح ويب (Chrome أو Edge).

* **الطريقة البرمجية (للمطورين):**
    1. افتح مجلد المشروع باستخدام برنامج **Visual Studio Code**.
    2. اضغط على زر **"Go Live"** في الشريط السفلي لتشغيل الموقع عبر إضافة **Live Server**.
    3. سيتم فتح الموقع على العنوان المحلي `http://127.0.0.1:5500`.

> **ملاحظة:** يتطلب التشغيل (سواء السحابي أو المحلي) وجود اتصال فعال بالإنترنت لضمان قدرة المنصة على الاتصال بـ **API نماذج عِلم** ومعالجة النصوص طبياً بشكل فوري.
